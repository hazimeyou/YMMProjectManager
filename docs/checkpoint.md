# チェックポイント運用

## 概要

チェックポイントは、後から見返したい大きな保存地点です。

保存内容:

* `.ymmp`
* `.ymmpx`
* サムネイル一覧
* 代表サムネイル
* 名前 / 説明 / コメント
* Git Commit / Git Branch
* サムネイル生成方式

## 世代管理との違い

* 世代管理: 軽量な `.ymmp` スナップショット
* チェックポイント: 復元・確認用のまとまった保存地点

## 作成方法

1. YMM で対象プロジェクトを開きます
2. プロジェクト一覧で対象を右クリックします
3. 次のいずれかを選びます
4. `チェックポイントを作成`
5. `現在開いているPFからチェックポイント作成`
6. 名前、説明、コメント、サムネイル生成方式を入力します
7. `作成` を押します

## 一覧 / 詳細

一覧では次を確認できます。

* 作成日時
* 名前
* 説明
* Git Commit / Branch
* 代表サムネイル
* サムネイル生成方式

詳細では次を確認できます。

* ymmp / ymmpx の保存先
* コメント
* サムネイル一覧
* 選択中サムネイルの拡大表示

## 診断

診断では次を確認します。

* manifest の存在
* manifest の読込
* ymmp の存在
* ymmpx の存在
* 代表サムネイルの存在
* サムネイル一覧の存在
* 復元に必要な主要ファイルの有無
* Git Commit / Git Branch の記録

判定:

* `OK`
* `Warning`
* `Error`

## 復元

1. チェックポイント一覧または詳細から `復元` を選びます
2. 復元先フォルダーを指定します
3. 確認ダイアログを確認して実行します

復元は既存プロジェクトへ直接上書きしません。

## 削除

1. チェックポイント一覧または詳細から `削除` を選びます
2. 名前、日時、保存先を確認します
3. 確認ダイアログで実行します

削除対象:

* manifest 上の該当項目
* チェックポイント専用フォルダー
* `ymmp`
* `ymmpx`
* 代表サムネイル
* サムネイル一覧

元プロジェクトは削除しません。

## 既知の制限

* サムネイル生成は、YMM の現在のプレビュー取得に依存します
* 現在開いている YMM プロジェクトを取得できない場合、専用導線は使えません
* Git 情報は Git リポジトリ外では空になる場合があります
