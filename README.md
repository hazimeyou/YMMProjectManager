# YMM Project Manager

YukkuriMovieMaker 用の補助ツールです。

本ツールは以下の機能を提供します。

* 素材再リンク（自動探索）
* サムネイル生成
* チェックポイント管理（ymmpx / サムネイル / Git情報付き保存・復元）
* 世代管理（.ymmp のローカルスナップショット保存・復元）
* `ymmpx` 同梱作成
* `ymmpx` 展開

---

# インストール方法

1. GitHub の Release ページから `YMMProjectManager.ymme` ファイルをダウンロードします
2. ダウンロードした `YMMProjectManager.ymme` を起動します
3. YukkuriMovieMaker の「ツール」タブからプロジェクトマネージャーを実行します

---

# 主な機能

## 素材再リンク

.ymmp ファイル内の素材（画像 / 動画 / 音声）のリンク切れを検出し、自動探索で修復します。

## プロジェクト一覧表示

.ymmp ファイルを登録してプロジェクトの一覧を表示します。登録したプロジェクトはサムネイル付きで管理できます。

一覧の右クリックメニューから次の操作ができます。

* チェックポイントを作成
* チェックポイント一覧
* 世代を保存
* 世代一覧
* 世代診断

## チェックポイント

v0.5.0 でチェックポイント機能を追加しました。

チェックポイントは通常の世代保存より大きな保存地点として扱います。保存時に次の情報をまとめて記録します。

* `ymmp`
* `ymmpx`
* サムネイル一覧
* 代表サムネイル
* 名前 / 説明 / コメント
* Git Commit / Git Branch
* サムネイル生成方式

通常の世代管理との違い:

* 世代管理は `.ymmp` のローカルスナップショット保存が中心です
* チェックポイントは `ymmpx`、サムネイル、Git情報もまとめて残す大きな保存地点です
* 世代管理は軽量な戻し先、チェックポイントは後から振り返る節目、という使い分けを想定しています

### 作り方

1. 対象プロジェクトを YMM で開きます
2. プロジェクト一覧で同じ `.ymmp` を選びます
3. 右クリックメニューの `チェックポイントを作成` を押します
4. 名前、説明、サムネイル生成方式を選んで保存します

現在 YMM で開いているプロジェクトから直接作りたい場合は、右クリックメニューの `現在開いているPFからチェックポイント作成` を使います。

注意:

* サムネイル生成は開いているタイムラインのプレビューを使います
* 現在は、選択中プロジェクトと YMM で開いているプロジェクトが一致している必要があります
* 生成方式は `均等分割` / `1秒ごと` / `5秒ごと` / `10秒ごと` / `30秒ごと` / `1分ごと` / `5分ごと` / `任意秒数` / `任意分数` に対応しています

### 一覧 / 詳細

`チェックポイント一覧` では次の情報を確認できます。

* 名前
* 作成日時
* 説明
* 代表サムネイル
* Git Commit
* Git Branch
* サムネイル生成方式

詳細画面では `ymmpx` 情報、コメント、サムネイル一覧、拡大表示を確認できます。詳細画面から診断と削除も実行できます。

### 削除

`チェックポイント一覧` または `チェックポイント詳細` から削除できます。

* 削除前に確認ダイアログを表示します
* 削除対象の名前、日時、保存先を確認できます
* 元の `.ymmp` プロジェクトは削除しません

### 診断

`チェックポイント一覧` または `チェックポイント詳細` から診断できます。

診断では次を確認します。

* manifest の存在 / 読込可否
* ymmp / ymmpx の存在
* 代表サムネイルとサムネイル一覧の存在
* 復元に必要な主要ファイルの有無
* Git Commit / Git Branch の記録有無

判定は `OK` / `Warning` / `Error` で表示します。

### 復元

チェックポイント復元は、既存プロジェクトへ直接上書きせず、指定フォルダーへ `ymmpx` を展開します。

保存先は `%AppData%\\YMMProjectManager\\Checkpoints\\` です。

詳細な運用手順は [docs/checkpoint.md](docs/checkpoint.md) を参照してください。

世代管理は Git を使わず、`%AppData%\YMMProjectManager\Generations\` にローカル保存します。

保存対象は `.ymmp` 本体のみです。画像、動画、音声、フォント、素材フォルダは保存しません。

復元時は既定で元ファイルへ戻し、復元前バックアップを作成します。

## 世代診断

`世代診断` では次の情報を確認できます。

* ProjectId
* ProjectPath
* GenerationCount
* StorageSize
* LatestGeneration
* ManifestStatus
* DeletedGenerationCount

破損時の動作:

* manifest 破損時は空一覧として扱い、ログに記録します
* metadata 破損時は壊れた世代だけを無効化し、残りは表示します
* SHA256 不一致時は復元を禁止します

## ymmpx 同梱・展開

v0.4.0 で `ymmpx` 同梱・展開に対応しました。

プロジェクト一覧の上部にある `同梱` / `開いているPFを同梱` / `ymmpx展開` から操作できます。

### 同梱の作り方

1. 一覧で対象の `.ymmp` を選びます
2. `同梱` を押して保存先を指定します
3. もしくは YMM4 で編集中のプロジェクトを `開いているPFを同梱` で保存します

同梱ファイルは `YMMResourcePackager` と同じ `.ymmpx` 仕様を使います。`YmmpxLibPlugin` が必要です。

### 展開の仕方

1. `ymmpx展開` を押します
2. 展開したい `.ymmpx` と展開先フォルダーを選びます
3. 展開後の `.ymmp` と素材が出力されます

展開後は素材パスが展開先の実ファイルへ復元されます。

### 補足

* `ymmpx` 同梱・展開には `YmmpxLibPlugin` が必要です
* `YMMProjectManager` と `YMMResourcePackager` は同じ `.ymmpx` 仕様を使います
* `YmmpxLib` が見つからない場合は、同梱・展開は実行できません
* `YmmpxLib.dll` はリポジトリに同梱していません
* `YmmpxLibPlugin` は `https://github.com/hazimeyou/YmmpxLib/releases/latest/download/YmmpxLibPlugin.ymme` から取得されます
* `YmmpxLibPlugin` の導入後は YMM4 の再起動が必要です
* 未導入時は YMMProjectManager 上で導入案内が表示され、`YmmpxLibPlugin.ymme` を自動ダウンロードして起動できます
* `.ymmpx` のダブルクリック展開や Windows 関連付けは、現時点では対象外です

### 既知の制限

* 実際の同梱・展開には、YMM4 環境内で `YmmpxLibPlugin` が解決できる必要があります
* 壊れた `.ymmpx` や不正な `.ymmp` は、短い日本語メッセージとログでエラーを通知します
* 高度な除外設定 UI や独立した unpacker は、`YMMResourcePackager` 側の機能をそのままは持ち込みません

## 再リンクの使い方

1. プロジェクトを開きます
2. `開いているPFを再リンク`をクリックして、素材再リンクウインドウを開きます
3. `ファイル検索`をクリックして、素材の再リンク処理を開始します
4. 結果を確認して、必要に応じて候補を選択して修正します
5. `更新内容を保存`をクリックして、プロジェクトを更新します
6. 自動でタイムライン上の素材リンクが更新され、再リンクされます

---

## サムネイル生成

プロジェクトのプレビューからサムネイルを生成します。

* 64点サンプリング
* 非同期処理
* UI操作に影響を与えない設計

注意：

* YMM の表示状態に依存します
* プレビューが更新されない場合は取得できないことがあります
